Model K Volatility Decomposition

This directory contains five segment-specific copies of the usual Model K outputs, split by hourly BTC realized volatility measured from Binance 1-minute candles aligned to each scored Kalshi forecast hour.

Realized Volatility Definition

Hourly realized volatility is sqrt(sum(log(close/open)^2)) over the 60 Binance 1-minute bars in the forecast hour. Segment thresholds are computed on unique scored hourly Kalshi markets, not on individual minute-level forecast rows.

Shared Root Files

  • binance_1m_klines.csv: cached Binance minute data used for the decomposition.
  • binance_hourly_realized_volatility.csv: realized volatility per Binance hour.
  • hourly_market_volatility_segments.csv: each scored Kalshi hour with its realized volatility and segment flags.
  • segment_thresholds.csv: the quantile cutoffs used in this run.
  • all_scored_rows_with_volatility.csv: scored Model K rows with attached hourly volatility metadata.
  • kalshi_official_outcomes_long.csv: official contract outcomes used for all segment runs.
