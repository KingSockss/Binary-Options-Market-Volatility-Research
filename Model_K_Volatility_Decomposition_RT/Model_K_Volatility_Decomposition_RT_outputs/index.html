Model K Real-Time Volatility Decomposition

This directory contains five segment-specific copies of the usual Model K outputs, split by hourly BTC realized volatility measured from Binance 1-minute candles and classified with a trailing 72-hour real-time window.

Real-Time Volatility Definition

Hourly realized volatility is sqrt(sum(log(close/open)^2)) over the 60 Binance 1-minute bars in the forecast hour. Each scored hour is then compared against the trailing 72 Binance hours (4320 minutes) available at that time, so the segment cutoffs move through time instead of using one full-sample threshold for the entire run.

Shared Root Files

  • binance_1m_klines.csv: cached Binance minute data used for the decomposition.
  • binance_hourly_realized_volatility.csv: realized volatility per Binance hour plus rolling real-time thresholds.
  • hourly_market_volatility_segments.csv: each scored Kalshi hour with its realized volatility, rolling thresholds, and segment flags.
  • segment_thresholds.csv: summary statistics for the rolling percentile cutoffs used in this run.
  • all_scored_rows_with_volatility.csv: scored Model K rows with attached hourly volatility metadata.
  • kalshi_official_outcomes_long.csv: official contract outcomes used for all segment runs.
